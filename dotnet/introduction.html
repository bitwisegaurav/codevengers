Introduction to .NET Framework

The .NET Framework is a software development platform developed by Microsoft. It provides a programming model, a comprehensive set of libraries, and a runtime environment for creating and running applications.

With .NET, developers can build a wide range of applications, including web applications, desktop applications, mobile apps, gaming applications, and more.

One of the key features of .NET is its language independence. It supports multiple programming languages such as C#, VB.NET, F#, and more, allowing developers to choose the language they are most comfortable with. These languages are called .NET compliant languages.

.NET also includes a rich set of libraries and frameworks, such as ASP.NET for web development, Windows Forms and WPF for desktop applications, and Xamarin for cross-platform mobile app development.

Overall, .NET provides a powerful and versatile platform for building modern, scalable, and secure applications.

Architecture and Components

It is considered an engine for providing runtime services using its components like Common Language Runtime (CLR) and .NET Framework Class Library (FCL).

The main components of .NET Framework are :

  • CLR
  • FCL

Common Language Runtime (CLR)

CLR is the virtual machine which provides runtime environment for all types of .NET applications.

The source code is compiled by that language's compiler into CLR's intermediate language called MSIL (MicroSoft Intermediate Language).

CLR also manages memory, threads, exceptions, code exection, code safety, verification and compilation.

MSIL (MicroSoft Intermediate Language) or IL (Intermediate Language)

MSIL is a low-level, platform-independent representation of the code that can be executed by the CLR. It is comparable to bytecode in Java.

Working of CLR

  • When a .NET application is executed, the .NET Language Compiler compiles it to MSIL.
  • Then JIT (Just In Time) compiler component of CLR converts MSIL code into native code of machine.

Components of CLR

  • Common Type System (CTS)
  • Common Language Specification (CLS)
  • Garbage Collector (GC)
  • Just in time compiler (JIT)
  • Metadata and Assemblies

Common Type System (CTS)

It provides guidelines for declaring, using and managing data types at runtime. It offers cross language communication. It provides two categories of types :

  • Value Types : It stores the data in memory allocated on stack or inline in a structure. It holds data directly and two variables store data independent after copy data.
  • Reference Types : It stores the reference/address to the value of memory address and is allocated on heap.

Common Language Specification (CLS)

It is a subset of CTS which all .NET languages are expected to follow. It is a set of rules that needs to be followed while writing code in any .NET language to ensure that the code can be used by any other .NET language.

Garbage Collector (GC)

It is a CLR component which automatically manages memory. It is responsible for allocating and freeing memory for objects. It runs in the background and keeps track of objects that are no longer being used by the application.

Just in time compiler (JIT)

It is a CLR component which converts MSIL code into native code that can be executed by the CPU. It compiles the MSIL code into native code on demand at runtime.

There are three types of JIT compilers :

  • Economical JIT : It compiles only those methods which are called at runtime.
  • Normal JIT : It also compiles only those methods which are required at runtime but places them in cache for future use.
  • Pre-JIT : It compiles all methods at compile time.

Metadata and Assemblies

Metadata is the data about the data. It is binary information stored in .NET assemblies. It contains information about the types, members, and references in the code. It is used by the CLR for type safety, security, and interoperability.

.NET Framework Class Library (FCL)

It is comparable to Java Foundatoin Classes. This library contains numerous pre written and resuable types such as classes, interfaces, structures and enumerated values. Its types are grouped into namespace so that they can be accessed easily.

Managed and Unmanaged Code

  • Managed Code : The code created by compiler of any of the .NET Framework languages is called managed code. It is in MSIL so cannot be run directly, but runs under CLR.
  • Unmanaged Code : It is the code that is executed outside the CLR. It is compiled to machine code and runs under process of Operating System.

Advantages of .NET Framework

  • Object Oriented : A programming paradigm that organizes data and behavior into reusable structures called objects.
  • Cross Languages Interoperability : The ability of different programming languages to seamlessly work together within the .NET Framework.
  • Developer Productivity : The efficiency and effectiveness of developers in creating software using the .NET Framework.
  • Performance : The speed and efficiency at which software applications built with the .NET Framework execute.
  • Multiple Application Development : The capability of the .NET Framework to support the development of various types of applications.
  • Security : The measures and features provided by the .NET Framework to protect software applications from unauthorized access and attacks.
  • Code Robustness : The ability of software applications built with the .NET Framework to handle errors and exceptions gracefully.
  • Backward Compatibility : The ability of newer versions of the .NET Framework to run applications built with older versions without requiring major modifications.
  • Huge Library of Code : The extensive collection of pre-built code and libraries available in the .NET Framework for developers to use and leverage.
  • Intellisense : An intelligent code completion feature in integrated development environments (IDEs) that provides suggestions and auto-completion for code.

Disadvantages of .NET Framework

  • Managed code may be slwoer than native code.
  • Migration of application to .NET Framework can be expensive.
  • It is provided by Microsoft and it involves the danger of vendor lock-in.
