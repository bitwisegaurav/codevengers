Functions and Procedures

Procedures

Procedures are a block of code that performs a specific task. It's a self-contained block of statements that can be called from other parts of the code. The procedure does not return a value to the calling code.

Functions

Functions are procedures that perform a specific task and return a value to the calling code.

Built in Functions in VB.NET

VB.Net provides a large number of built-in functions that can be used to perform a specific task. Some of the built-in functions are:

  • Mathematical Functions
  • String Functions
  • Date and Time Functions
  • Type Conversion Functions
  • Format Functions
  • User Input Functions

Creating a Function

[Modifiers] Function FunctionName([Parameter List]) As ReturnType
    'Statements
End Function

Calling a Function

Dim result As ReturnType
result = FunctionName([Arguments])

Sub Procedures

A sub-procedure is a procedure that's called from another procedure or function. It's a way of breaking down a larger procedure into smaller, more manageable parts Sub procedures are defined using the Sub keyword.

Syntax

[Modifiers] Sub ProcedureName([Parameter List])
    'Statements
End Sub

Parameters and Arguments

Parameters are variables that are used to pass values to a procedure or function. Arguments are the actual values that are passed to the procedure or function.

Passing Parameters by Value

When you pass a parameter by value, a copy of the value is passed to the procedure or function. Any changes made to the parameter inside the procedure or function will not affect the original value.

Passing Parameters by Reference

When you pass a parameter by reference, the reference of the actual value is passed which make all the changes to the original value.

Syntax

Sub AddNumbers(ByVal num1 As Integer, ByVal num2 As Integer)
Dim result As Integer
result = num1 + num2
Console.WriteLine("The sum of {0} and {1} is {2}", num1, num2, result)
End Sub
