Exception Handling in VB.NET

Types of Errors

There are three types of errors in VB.NET :

  • Compile-time errors : These errors occur at the time of compilation of the code. For example, syntax errors.
  • Runtime errors : These errors occur at the time of execution of the code. For example, division by zero, array index out of bound, etc.
  • Logical errors : These errors occur when the logic of the code is not as per the requirement. It is also known as a bug.

Exception Handling

Exception handling is a mechanism to handle runtime errors. It is mainly used to handle the checked exceptions. If an exception occurs in the code, the rest of the code will not execute. So, it is recommended to handle the exception.

In VB.NET, exceptions are handled using try catch block

These blocks are :

  • Try : It contains the code that may throw an exception.
  • Catch : It contains the code that will execute if an exception occurs in the try block.
  • Finally : It contains the code that will execute whether an exception occurs or not.

Syntax : 

Try
    'Code that may throw an exception
Catch ex As [Exception]
    'Code to handle the exception
Finally
    'Code that will execute whether an exception occurs or not
End Try

Exception Class

Exception class is the base class for all exceptions in VB.NET. It is present in the System namespace. It has the following properties and methods :

  • Message : It contains the message of the exception.
  • StackTrace : It contains the stack trace of the exception.
  • InnerException : It contains the inner exception of the exception.
  • ToString : This method converts the exception name, description and current stack dump into a single string.

Some common exceptions

  • System.Exception : It is the base class for all exceptions.
  • System.ApplicationException : It is the base class for all application-specific exceptions.
  • System.SystemException : It is the base class for all system exceptions.
  • System.IndexOutOfRangeException : It is thrown when an array is indexed with a value less than zero or greater than the length of the array.
  • System.NullReferenceException : It is thrown when an object is not initialized.
  • System.DivideByZeroException : It is thrown when a number is divided by zero.

Conclusion

Exception handling is a mechanism to handle runtime errors. It is mainly used to handle the checked exceptions. In VB.NET, exceptions are handled using try catch block. The exception class is the base class for all exceptions in VB.NET. It has the properties and methods to handle the exception.
